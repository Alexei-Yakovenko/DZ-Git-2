Семантичні теги

Семантичні теги потрібні для описання смислового значення коду або контенту з яким він застосовується. Це потрібно для кращої ідентифікації користувачем, пошуковим роботом і браузером.

Приклади семантичний тегів

  • header - використовується для заголовкових або навігаційних елементів
  • main - для позначення основної частини сайту
  • nav - визначає, групу посилань, що є навігацією по сайту
  • article - позначає важливі розділи контенту всередині веб-сторінки
  • aside - для вміщення контенту, який є відокремленим від основного контенту сторінки
  • details - вказує додаткові відомості які користувач може за вимогою приховати або переглянути
  • figcaption - використовується для пояснення тегу "figure"
  • figure - викрпистовується дял позначення контенту, який щось ілюструє, анприклад для зображення або відео
  • section - універсальний тег для розділу документа на секції
  • summary - вказує, що контент являється науковою роботою, звітом, або іншим документом подібного значення
  • time - використовується для застосування дати чи календаря
  • footer - вказує на "підвал" сайту
